# 스파이어 어센트 (Spire Ascent)

> **Slay the Spire**에서 영감을 받은 브라우저 로그라이크 덱 빌딩 게임 — 팬 메이드 오마주.
> 모든 코드와 아트(CSS/SVG)는 오리지널로 새로 제작했습니다. MegaCrit과 무관한 비상업 팬 프로젝트입니다.

## 게임 소개

철갑 전사가 되어 **3개의 막(각 15층 + 보스)** 으로 이루어진 첨탑을 끝까지 등반합니다.
매 턴 에너지 3으로 카드를 사용해 적의 **의도**를 읽고 공격과 방어를 저울질하며,
전투 보상으로 덱을 키워 마지막 보스를 쓰러뜨리면 **엔딩**을 볼 수 있습니다.

| 콘텐츠 | 규모 |
|---|---|
| 카드 (전사 클래스) | 37종 — 기본/일반/특별/희귀 + 상태이상/저주, 전 카드 강화(+) 지원 |
| 적 | **53종** — 1막(유충·슬라임·도깨비 무리…) / 2막(구체 파수병·선택받은 자·백부장과 신비술사…) / 3막(어둠덩이·아귀·일렁이는 형상…) |
| 정예 | 막별 3종 × 3막 = 9종 (도깨비 거두 · 잠자는 갑충 · 파수꾼 / 찌르기 전서 · 도깨비 대장 · 스네코 / 거대한 두상 · 응보자 · 파충류 술사) |
| 보스 | 막별 3종 중 1종 — 거대 수호자(모드 전환) · 육염령(화염 주기) · 슬라임 대왕(분열) / 챔피언(분노) · 청동 자동인형(구체 소환) · 수집가(소환+대저주) / **각성자(부활)** · **시간 포식자(회복)** · **도누&데카(2체)** |
| 보스 메커니즘 | 분열 · 소환 · 부활(2페이즈) · 무형(피해 1 고정) · 방벽(방어도 유지) · 자폭 · 도주 |
| 유물 | 16종 (끓는 피, 닻, 등불, 펜촉, 쿠나이 …) |
| 물약 | 10종, 3슬롯, 동적 드랍률(40%±10%) |
| 지도 | 막마다 7열 × 15층 절차 생성, 분기 경로, 9층 보물 / 15층 휴식처 고정 |
| 방 종류 | 몬스터 · 정예 · 미지의 장소(이벤트 6종) · 상인 · 보물 · 휴식처 · 보스 |

핵심 전투 규칙은 원작의 수치를 충실히 따랐습니다: 취약 +50% / 약화 -25% / 손상 -25%(내림),
방어도는 턴 시작 시 소멸, 광신도 의식 3, 거대 수호자 모드 전환 30(+10씩 증가) 등.
2막부터는 보상 카드가 일정 확률(25%/50%)로 강화되어 등장합니다. 자세한 조사 내용은 [`docs/`](docs/)를 참고하세요.

## 조작

- **드래그 앤 드랍**: 카드를 끌어서 사용합니다.
  - 단일 대상 카드(타격 등) → **적 위에 드랍** (조준 화살표가 표시되고, 적 위에서 초록색으로 바뀝니다)
  - 그 외 카드(수비·파워·광역기) → **위쪽 전장으로 꺼내서 드랍**
- **숫자키 1~9, 0**: 손패 순서대로 카드를 집습니다. 집은 카드는 마우스를 따라오며, **클릭으로 드랍**합니다. (같은 숫자키/ESC/우클릭 → 내려놓기)
- **E** 턴 종료 · 적 머리 위 아이콘에 마우스를 올리면 의도 설명
- 좌하단 에너지 구슬 / 뽑을 더미, 우하단 버린 더미 / 소멸 더미 (클릭하여 내용 확인)
- 진행 상황은 자동 저장됩니다 (localStorage)

## 로컬 실행

ES 모듈을 사용하므로 정적 서버가 필요합니다:

```bash
python3 -m http.server 8000   # 또는 npx serve
# http://localhost:8000 접속
```

## 테스트

DOM 없이 동작하는 전투 엔진을 노드에서 직접 검증합니다:

```bash
node test/smoke.mjs
```

- 카드/적/유물/이벤트 데이터 무결성 + 전 막 조우 테이블 유효성
- 지도 생성 300시드 (연결성·고정층·금지 규칙)
- 피해 공식 (취약/약화 내림 처리)
- 메커니즘 단위 검증 (모드 전환, 분열, 결계, 무형, 방벽, 부활, 소환, 도주, 시간 역행 회복 …)
- 1~3막 전 조우(보스 포함) × 20시드 = **900회 무작위 플레이 시뮬레이션** (예외·무한루프 0건)
- 1막 → 3막 → 엔딩 전체 런 플로우 검증

브라우저 레벨 테스트(헤드리스 크롬):

- `test/e2e-drag.html` — **실제 포인터 이벤트**로 드래그 앤 드랍/숫자키/턴 진행을 구동하는 E2E (17개 검증)
- `test/ui-harness.html` — 전투/지도 화면 DOM 렌더링 점검
- `test/shots.html?s=rest|ending|trans|reward|…` — 화면별 시각 QA용 스크린샷 하네스

## 구조

```
index.html          진입점 (빌드 불필요)
css/style.css       StS 스타일 다크 판타지 UI (오리지널 CSS)
js/
  main.js           런 상태 머신 · 저장/불러오기
  combat.js         전투 엔진 (DOM 없음 — 노드에서 테스트 가능)
  combatui.js       전투 화면 (의도/타게팅 화살표/핸드 부채꼴/연출)
  screens.js        타이틀·지도·보상·휴식·상점·이벤트·보물·게임오버
  map.js            7×15 절차 지도 생성
  ui.js             공용 위젯 (카드·툴팁·상단바·오버레이)
  art.js            오리지널 SVG 스프라이트/아이콘
  data/             카드·적·유물·물약·이벤트 데이터
test/               스모크 테스트 + UI 하네스
docs/               Slay the Spire 심층 조사 자료
```
